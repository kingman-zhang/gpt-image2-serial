# gpt-image2-serial 用户级 API 配置设计

## 背景

当前 skill 支持环境变量和项目根目录下的 `.env.image`。这种方式要求用户理解 shell 环境变量、项目根目录以及示例文件的位置，对第一次使用 Agent skill 的用户不够友好。

通过 skills installer 安装时，仓库根目录的 `.env.image.example` 不会出现在用户当前项目中。因此，直接提示用户执行 `cp .env.image.example .env.image` 可能失败。

同时，不能要求用户把 API key 粘贴到聊天中。聊天记录、执行日志和 Codex shell 快照都可能长期保留敏感信息。

## 目标

- 用户只配置一次，即可在所有项目中使用该 skill。
- 缺少配置时，由 Agent 主动启动配置流程。
- API key 不进入聊天记录，也不在终端中回显。
- 用户无需理解“项目根目录”即可完成首次配置。
- 保留环境变量和项目级 `.env.image`，供高级用户覆盖。

## 非目标

- 不提供云端密钥托管。
- 不集成 macOS Keychain、1Password 等外部密钥管理器。
- 不自动获取或验证用户的 API key 来源。
- 不在安装阶段强制配置；配置在首次使用或用户主动请求时进行。

## 配置位置

默认用户级配置文件为：

```text
~/.config/gpt-image2-serial/env
```

配置文件使用 dotenv 格式：

```dotenv
OPENAI_IMAGE_API_KEY=...
OPENAI_IMAGE_BASE_URL=https://api.openai.com/v1
```

配置目录不存在时自动创建。配置文件写入后设置为 `0600`，只允许当前用户读写。

## 配置优先级

生成脚本按以下顺序解析配置，前者覆盖后者：

1. 当前进程环境变量 `OPENAI_IMAGE_API_KEY`、`OPENAI_IMAGE_BASE_URL`
2. 兼容环境变量 `OPENAI_API_KEY`、`OPENAI_BASE_URL`
3. 当前工作目录的项目级 `.env.image`
4. 用户级 `~/.config/gpt-image2-serial/env`
5. Base URL 默认值 `https://api.openai.com/v1`

项目级配置用于有隔离需求的高级用户。用户级配置是 README 和 Agent 首次引导中的默认方案。

## 安全配置向导

skill 新增 `scripts/configure.py`，只使用 Python 标准库。

运行方式：

```bash
python3 /path/to/gpt-image2-serial/scripts/configure.py
```

交互流程：

1. 显示即将写入的完整配置路径。
2. 使用 `getpass.getpass()` 隐藏输入 API key。
3. 提示输入 Base URL，直接回车时使用 OpenAI 默认地址。
4. 将配置先写入同目录临时文件，再原子替换目标文件。
5. 设置目标文件权限为 `0600`。
6. 只输出配置成功、配置路径和下一步，不输出 key。

如果配置文件已存在，向导先询问是否更新。用户拒绝时不修改文件。非交互终端中不尝试读取 key，也不接受命令行 `--api-key` 参数，避免密钥进入 shell history 和进程列表。

## Agent 行为

`SKILL.md` 的首次使用流程调整为：

1. 检查环境变量、项目级配置和用户级配置是否存在，不回显值。
2. 如果缺少 API key，告诉用户将启动安全配置向导，并显示配置路径。
3. 在可交互终端中运行 `configure.py`，让用户隐藏输入。
4. 配置成功后继续执行用户原来的图片生成请求，无需用户重新描述任务。
5. 如果当前 Agent 无法提供交互终端，明确说明限制，并指导用户在自己的终端运行一条完整命令来启动向导。

Agent 不得要求用户把 API key 发到聊天里，也不得通过命令参数、日志或截图传递密钥。

## 生成脚本行为

包装脚本继续作为唯一推荐入口。它负责加载配置，并将解析后的值传给独立 Python 客户端。

为了避免 `source` 执行配置文件中的任意 shell 代码，新增的用户级配置和现有项目级 `.env.image` 都应通过受限 dotenv 解析读取，只接受已知变量名和简单键值格式。

缺少配置时，错误信息应包含：

- 用户级配置文件的完整展开路径
- 启动配置向导的完整命令
- “不要把 API key 发到聊天里”的提醒

## 错误处理

- 空 API key：不写入配置，提示重新输入。
- 非法 Base URL：只接受 `http://` 或 `https://`，不写入配置。
- 无法创建目录或写入文件：报告目标路径和系统错误，不输出敏感值。
- 无法设置 `0600`：视为配置失败，清理临时文件。
- 已有配置：未经确认不覆盖。
- 非交互环境：停止并给出用户可在本地终端执行的向导命令。

## 验收标准

- 新用户安装后，只需让 Agent 启动安全配置向导即可完成配置。
- 用户不需要知道项目根目录或 `.env.image.example` 的位置。
- API key 不经过聊天、命令行参数或终端回显。
- 配置一次后，在不同项目目录中均可生成图片。
- 现有环境变量和项目级配置仍然兼容。
